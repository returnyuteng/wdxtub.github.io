写一本书

发表于 2016-06-08   |   分类于 Technique   |     |  

努力去做好自己想要做的事情，才是通往梦想的捷径。

喜欢音乐，当然想要有一张属于自己的专辑；喜欢摄影，当然想要有一场属于自己的影展；喜欢运动，当然想要有一块属于自己的奖牌；喜欢写作，当然想要有一本属于自己的书。

话是这么说，真的有一天要我写一本书，一本教材的时候，第一感觉是『不真实』。回想自己的写作历程，起初瞎写，后来写写生活写写书评，最后回归本行写写技术，追根溯源一切的开端竟是一个小小的念头，不由感叹生活的奇妙。要是我当初没有在最后一个学期选工作量比较大的两门课，没有因为『想通过日志让更多的人感受美式教育』这个念头而开始写博客，没有顶住压力坚持写下去的话，恐怕现在又是另一番光景。

这次有机会执笔写一本关于计算机系统基础的教材，于我来说是特别好的机会，和特别大的挑战。终于能按照自己的想法去解决目前中文教材的诸多问题，终于能够做一件能影响到许多人（尤其是年轻人）的事，真的高兴能有这样的机会，这也是为什么我要义无反顾地回国，在这里，我能做得更多。但写书不比写博客，对文字、编排、配图都有更高的要求，作为初出茅庐的毛头小子，肯定少不了反复删改打磨，而且白纸黑字印出来就不能改了，所以写作的时候，一定要精细再精细，慎重再慎重。另外具体的写作节奏我也没有经验，写系列文章可以想一出是一出，但是写书，就得事先大概计划好。而且具体内容和图片也需要反复斟酌，有了之前若干次被『查水表』的经历，以后可不能再不把版权当回事儿了。

现在博客中的 读薄/读厚 CSAPP 系列 是在原先课程笔记的基础上重制的版本，重制的过程中对所学内容有了更深的理解。而这次写书，更是对于计算机系统相关知识的一次验收，如果能顺利完成，想必是大有裨益。正好也能借此机会向前辈和大牛们学习请教，是很好的起点。都说还有『诗与远方』，『诗』暂时算了，但是『远方』我是特别想去的。

正好高考结束，来说说教育的事儿吧。之前的日志中提到过，大环境总体来说还是比较浮躁和急功近利的，这需要时间来慢慢扭转。作为基础教育，语数英政物化史地生这么多门学科，编程应该也必须成为其中之一，这种训练对于帮助学生构建完整的逻辑和系统思维都有很大的帮助。在这个构建在数字之上的新世界，编程真的是基本素质，人类不断发展科技，科技反过来也在影响人类的进化。反观高校教育，不少计算机类教材和实际生活脱节，并且有着清晰的学科分野，缺少跨界和实践。唯一的例外是考研类书籍，年年更新，大行其道，很难想象如何能从中学到真正有用的东西。

我所能做的是结合自己在国内和国外学习的感受，挑真正有价值且学生最需要的东西来写。因为经验有限无法和经典相提并论，但我一定会拿出两百分的诚意，争取一切可能争取的资源，把这本书写好。

做一件事，是因为自己想做才去做，去思考如何能做得更好。这样的态度已经给我带来了太多惊喜，要一直坚持下去。这篇日志算是『揭幕仪式』，期待着这栋『大楼』能早日『封顶』，早日交付吧。

与诸君共勉。
