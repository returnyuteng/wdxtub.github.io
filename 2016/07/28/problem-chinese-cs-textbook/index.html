我们的中文计算机教材怎么了？

发表于 2016-07-28 | 更新于 2017-02-15 | 分类于 Technique | |

提到原创的中文计算机教材，我的心情是沉重的。这里是我对新时代教科书的一些思考。

更新历史

  • 2017.02.15: 修正错别字
  • 2016.12.20: 内容更新
  • 2016.07.28: 完成初稿

回顾自己在计算机领域『寒窗苦读』的七年，真正走上工作岗位的我，真正开始写基础课教材的我，才真正意识到我们的计算机教育和世界顶尖的差距，远不只是一系列课本这么简单。

问题所在：

  • 没有着力于激发兴趣，而是传统的填鸭式教育（虽然打着素质教育的大旗）
  • 课程设计没有与时俱进，毕竟短期的投入产出比太低（长期的太遥远我们常常忽视）
  • 利益绑定，老师出教材，强迫大家买书，强迫大家用 -> 高校教材『自留地』，大学体制问题
  • 重科研轻教学的『成果』导向政策
  • 课本的想成为学生的轮椅，而不是教会学生走路
  • 国内教材粗略，一带而过，可以轻松完成教学任务；国外教材详细，需要准备和课时都更多
  • 按工作量评比，但是质量不属于工作量
  • 喜欢用术语解释，不肯好好说话，原因在于老师自己对于某些概念可能都不甚明晰

而从课程体系的角度来看

  • 缺乏一门贯穿整个计算机系统的基础课程。计算机系统方面的知识被分成了很多门独立的课程，课程内容彼此之间缺乏关联和系统性
  • 现有课程往往偏重理论，和实践关联较少
  • 课程大多从设计者的角度触发，而不是从使用者的角度出发。而对大多数学生来说，最重要的是理解主流计算机系统的整体设计以及这些设计因素对于应用软件开发和运行的影响

针对这种情况，我给自己定的方向或者说目标是：

  • 贴近实际，结合工作中的感受，拉近学术界与工业界的距离
  • 在保证准确度的情况下，多用比喻让学生容易理解，也可以借此串起各个知识点
  • 使用类似打怪升级的模式，告诉学生能做什么、该怎么去做，保持学生兴趣
  • 结合各个领域的实践，指路，而不是帮助学生走路，引导为主
  • 培养学生的问题抽象、系统抽象和数据抽象能力

具体的写作思路大纲应该还会花一段时间来思考，但是现在至少有个方向了嘛。
